RCO example question

Posted on 2017-04-13   |   In Study Notes   |  

Question From: https://www.rco.recruit.co.jp/career/engineer/entry/

過去の出題例
Q.「相手の思考を推理する」過程をプログラムで表現してください。
※回答例はございません。予めご了承ください。
A,B,Cの3人が1～5の5枚のカードを使ってインディアンポーカーをします。
3人は、ランダムに1枚のカードを引いて額にかざします。相手のカードは見えますが、自分のカードは見えません。
この状態で、A->B->Cの順番に、自分が1番大きい（MAX）、自分は2番目に大きい（MID）自分が1番小さい（MIN）、わからない （?）、を答えます。
1人でも答えがわかった場合、そこで終了となります。「わからない」と答えた場合、回答権が次の人に移ります。 Cもわからない場合、再度Aに回答権が移ります。3人ともウソを言ったり、適当に答えてはいけません。
例1）「A=1 B=4 C=5」だった場合、「A => MIN」で終了します。
例2）「A=1 B=2 C=4」だった場合、「A => ?, B => MID」で終了します。
Bは「Aがわからないなら、自分は5ではない」と考えるからです。
以上を踏まえて、
引数で「A=1 B=4 C=5」で実行すると「A =>MIN」を出力
引数で「A=1 B=2 C=4」で実行すると「A =>?, B =>MID」
を出力するようなコマンドラインのプログラムを作成してください。
なお、人数やカードの枚数がパラメーター化されていて、さまざまなケースがシミュレーションできるコードが望ましいです。

前 提

  • 制限時間は1時間半
  • 今回のケースでは、「全員がわからない(無限ループ)」という結果にはなりません。

Q: Try to present the process of inference.
A, B, C use 5 cards with numbers of 1~5 respectively to play Indian Poker. Everyone draws one of the cards randomly and put it on his own forehead. The cards of others can be seen but one can’t see the card on his own forehead. Now they answer “MIN”, “MAX”, “MID” or “?” by order of A->B->C.

  • MIN: smallest in cards of A, B, C
  • MAX: biggest in cards of A, B, C
  • MID: between MIN and MAX
  • ?: don’t know
    If one answers MAX, MIN or MID, the game will be over. If one answers ‘?’, the next will answer. It will repeat until someone answers MAX MIN or MID. All of them will not tell a lie.
    e.g.1「A=1 B=4 C=5」
    「A => MIN」
    A thinks ‘I must be MIN for B and C are 4 and 5’.
    e.g.1「A=1 B=2 C=4」
    「A => ?, B => MID」
    B thinks ‘A didn’t know so I can’t be 5’.

It is desired that number of people and cards can be parameters and more kinds of circumstances can be simulated.

Constrains:

  • Time limit is 1.5 hours.
  • No such cases like “All answer ‘?’”(loop).

Here’s my solution and code in Python
First, simple 3 player version.
Just consider the process of inference.
A sees B, C and guesses his own possible numbers and the related answers of ‘MIN’ ‘MAX’ ‘MID’. If the answers are the same, he’ll answer it. If they are different, like ‘MIN’ ‘MAX’ ‘MIN’, he’ll answer ‘?’ for uncertainty.
B also guesses his possible numbers but then he will consider A’s inference first and remove the impossible guesses. For example of {A=1, B=2, C=4}, first A:’?’. B guesses ‘2, 3, 5’. He considers if {A=1, B=5, C=4}, A could answer MIN, so he can’t be 5. So he can only be 2 or 3, both are MID.
C also guesses but he considers both A and B and remove the impossible numbers then make the answer.
If it goes into a new round, they will also consider the answers of past rounds.

Second, the extended version.
Use recursion to validate if others’ answers are all ‘?’ given some guesses.

    What is Machine Learning?

    Illustration

    data -> learning machine -> knowledge

Definition

A computer program is said to learn from experience E with respect to some
class of tasks T and performance measure P, if its performance at tasks in T,
as measured by P, improves with experience E. — Tom M. Mitchell (1997)

Application

Document Categorization, Sentiment Analysis and Opinion Mining, Face Recognition
, Handwritten Character Recognition, Fingerprint Identification, Speech Recognition, Bioinformatics, Social Media Analysis, Stock Market Prediction, Many More…

  • Internet Search and Information Retrieval
  • Natural Language Understanding
  • Machine Translation
  • Locating/tracking/identifying objects in images & video
  • Financial prediction and Business Intelligence
  • Medical diagnosis, media image analysis
  • Recommendation Systems
  • Driverless Car
  • …

Machine Learning Subcategories

  • Supervised Learning: given examples of inputs and corresponding outputs, predict outputs on new inputs
    • Classification(Descrete labels), Regression(Continuous labels), etc.
  • Unsupervised Learning: given only inputs, automatically discover knowledge (labels, features, structure, etc.)
    • Clustering(Group similar patterns like words, images…), Density Estimation, etc.
  • Semi-supervised Learning
  • Ensemble Learning
  • Active Learning
  • Transfer Learning
  • Reinforcement Learning
  • Deep learning
  • …

Machine Learning Models

  • Sequential Model
  • Graphical Model
  • Discriminative Model
    • Model the posterior probability of label given observation p(y|x)
  • Generative Model
    • Model the joint probability of label and observation p(x, y), and then use the Bayes rule p(y|x)=p(x,y)/p(x) for prediction

Machine Learning Phases

  1. Training phase (using training data)
    • You present your data from your “gold standard” and train your model, by pairing the input with expected output
  2. Test phase (using test data)
    • In order to estimate how good your model has been trained, and to estimate model properties (such as mean error for regression, accuracy for classification)
  3. Application phase (for future data)
    • Now you apply your freshly-developed model to the realworld data and get the results

General Process

  1. Hypothesis
    • Math models with (unknown) parameters (or structures)
  2. Learning (to estimate the parameters)
    • Maximum Likelihood Estimation (MLE), MAP, Bayesian Estimation
    • Cost Function Optimization
  3. Decision
    • Bayes decision rule
    • Direct prediction function

Learning Criteria

  • Perceptron Criterion
  • Least Mean Square (LMS)
  • Minimum Cross Entropy (CE)
  • Maximum Margin Criterion
  • Maximum Likelihood
  • …

Optimization Methods

  • Analytic Solution
  • Gradient Descent
  • Stochastic Gradient Descent
  • Newton Method
  • Quasi-Newton Method (BFGS)
  • Limited Memory BFGS (L-BFGS)
  • Conjugate Gradient
  • GIS
  • IIS
  • …
